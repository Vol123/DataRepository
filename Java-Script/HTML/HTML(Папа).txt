--------------------- СТРУКТУРА HTML ДОКУМЕНТА --------------------

<!DOCTYPE html> 	- Непарный тег заголовка документа, html указазывает на версию Html 5.0
<html lang="LANG">	- ПАРНЫЙ тег указания на чтение броузером с языком LANG
	<head>		- заголовок документа html - информация для браузера и поисковика
		<title>	- ПАРНЫЙ тег-описание документа для поиска
		
		Непарные теги META

		<meta name="description" content="...text..."> 				- 2й уровень title для поиска
		<meta name="keywords" content="программирование, курсы, HTML, CSS"> 	- быстрый поиск по ключевым словам
		<meta charset="UTF-8"> 							- кодировка страницы для поиска
		<meta name="viewport" content="height=device-height"> или
		<meta name="viewport" content="width=device-width">			- прировнять ширину или высоту вывода к устройству, 
											чтобы не было горизонтальной прокрутки
					content="width=300" или
					content="height=500"				- точное указание размера 
		дополнительно:
		<meta name="viewport" content="initial-scale=0.5"			- масштаб страницы
		<meta name="viewport" content="user-scalable=yes"			- разрешение увеличивать/уменьшать масштаб страницы
			значения content можно перечислять через запятую, например:
				<meta name="viewport" content="width=device-width, initial-scale=1.0">
	</head>
	<body>		- тело документа html
		...
		...
	</body>
</html>

--------------------- МЕДИА --------------------

<img src="filesrc" alt="...text..." width="100" hieght="100">
	Непарный тег для вывода картинки filesrc с размерами width height, 
	а если картинка не загрузилась, 
	то вывести текст alt 

	атрибуты src= и alt= обязательные

<audio src="filesrc" controls></audio>
	Парный тег!!! для воспроизведения айдио файла filesrc. Controls - элементы плейера; для каждого броузера свои.
<source src="filesrc">
	Непарный вложенный тег для audio, чтобы перечислить варианты выбора filesrc, если формат не поддерживается броузером, например:
		<audio controls>
  			<source src="https://example.com/audio.mp3">
 		 	<source src="https://example.com/audio.ogg">
		</audio>

<video src="filesrc" controls></video>
	controls 	— Добавляет элементы управления для видеоплеера
	autoplay 	— Автоматическое воспроизведение после загрузки видео
	width 		— Ширина видеоплеера
	height 		— Высота видеоплеера
<source src="filesrc">
	Непарный вложенный тег для video, чтобы перечислить варианты выбора filesrc

--------------------- ФОРМЫ --------------------

Создание формы, где результат обработки будет находиться на странице "/Page". Если action не указано, то на текущей странице
<form action="/Page">
  <input type="textbox">	// Непарный тег <input> с аттрибутом type (разные контролы) - здесь это поле ввода
</form>

<form>
  <label>Введите имя					// Парный тег <label> для поля ввода ниже
      <input type="text" placeholder="Mask">		// аттрибут placeholder задает подсказку Mask в поле
  </label>			
</form>

<label>							
    <input type="checkbox" name="languages" value="HTML">	// Checkbox объединяемый в группу name= со значением выбора value= 
    Хочу изучать HTML
</label>

<input id="yes" type="radio" name="question" value="Yes">	// Radiobutton объединяемый в группу name= со значением выбора value=
<label for="yes">Да</label>					// связь с radio через id=

<textarea rows="5" cols="30">Начальный текст</textarea>		// многострочный ввод на 5 строк и 30 символов в строке

<select>						// выпадающий список выбора! Если <select size="5">, 
							// то выпадающий превратится в фиксированный на 5 строк
   <option disabled>Какой курс изучим?</option>		// disabled отключает выбор элемента
   <option value="JS">Супер язык JS</option>		// на сервер будет отправлен выбор JS, а не 'Супер язык JS' 
   <option>PHP</option>
   <option>Java</option>
</select>

<select multiple>		// список множественного выбора
   <option>JS</option>
   <option>PHP</option>
   <option>Java</option>
</select>

<button>Применить</button>	// Данные будут переданы обработчику, а !!! фоорма перезагружена

--------------------- ОБЛАСТИ --------------------

<main>
  Основной контент страницы
	Элемент main является уникальным и единственным для одной страницы. 
	При оборачивании контента в main важно не включать в него неуникальные области:
		Боковое меню
		Общую навигацию для сайта
		Общую шапку и футер сайта
		sections и arcticles
	Логотип
</main>

<h1>
  Основной заголовок (желательно один на страницу)
</h1>
<h2> ... <h6> 
  Заголовки по уровням ниже h1 (может быть несколько на страницу)
</h2> ... </h6> 

<article>
  Область не привязана структурно к конкретной странице. Например, блок с курсом валют, погода, превью новости,...
</article>

<section>
  Область связанных, по смыслу, которую нельзя отделить от страницы по смыслу
</section>

<header>
  Область шапки (вводная часть раздела article или section - для описания, анансов, навигации)
</header>

<footer>
  Область подвала страницы, как правило, не меняется, на разных страницах одного сайта,
	например лого, копирайты, ссылки и контакты, и т.п.
</footer>

<nav> 
    // Область различных меню 
    <!-- Меню -->
    <ul>
      <li><a href="/intro">Intro</a></li>
      <li><a href="/about">About</a></li>
    </ul>
</nav>

<aside>
  Область сайдбара или боковой панели.
	Служит для отображения информации, 
	которая дополняет основной текст, 
	но не является обязательной. 
	Чаще всего это меню раздела, список определений из статьи, виджеты и т.д.
</aside>

<div>
  Универсальный контейнер без логической составляющей, который до появления тегов сверху служил их заменой
</div>

<span>
  По смыслу как и div, но в него другие контейнеры уже не вкладывают. 
</span>
